# Official capability model (agency target state)

Provided by the user 2026-07-13, verbatim from the agency's working capability inventory (ESS/Eurostat-style model — note "ESS" in 3.3). This is the **intended seed data for the v2 taxonomy swap** (see CLAUDE.md "Direction — v2 reframing"): official L1/L2/L3 capabilities become the formal layer; the tool's tangible workbook processes map to leaf nodes and roll up.

Two columns matter beyond the hierarchy: **Proposed Business Owner** (the org has already decided who *should* own each capability — so overlap findings should be framed as deviation-from-intended-owner / migration candidates, not raw duplication) and **Product family** (the shared-service landing zone; sparse, only some rows have one).
